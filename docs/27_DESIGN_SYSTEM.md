# Design System

Version: 1.0

Status: Product Design

---

# Purpose

Define a unified visual language for EduPath.

Every page should feel like it belongs to the same product.

The design should communicate clarity, intelligence, trust, and growth.

---

# Design Philosophy

EduPath should feel:

• Intelligent

• Calm

• Modern

• Motivating

• Minimal

• Premium

Students should feel guided, not overwhelmed.

---

# Visual Inspiration

Reference products:

* Notion (clean layouts)
* Linear (minimalism)
* Duolingo (engaging feedback)
* GitHub (clear information hierarchy)
* Apple (spacing and typography)

---

Primary

---

Tertiary

## Charts

Minimal

Interactive

## Icons

Use a consistent icon set.

Avoid mixing styles.

Icons should reinforce meaning rather than decorate.

Celebrate:

Rank increases

# Empty States

Every empty state should educate.

Example:

"No Learning Insights yet.

Complete today's Knowledge Recovery to begin building your learning profile."

---

# Brand Principles

Every interface should answer three questions immediately:

1. What should I do today?

2. Why should I do it?

3. How will it improve my learning?

If a screen cannot answer these questions, simplify it.
