Purpose

Defines how EduPath will maximize its score in the Kaggle × Google Vibe Coding Capstone by aligning product decisions, engineering effort, and demo strategy with the judging rubric.

Overview

The competition evaluates submissions on:

Pitch (30%) – clarity of problem, value proposition, impact.
Implementation (70%) – technical quality, AI integration, architecture, code quality.

Our strategy is to optimize for implementation without sacrificing a compelling narrative.

Objectives
Build a technically rich but focused MVP.
Showcase at least three Google ADK concepts (multi-agent orchestration, session/state management, long-term memory, context engineering).
Create memorable "wow moments" during the demo.
Deploy a working application.
Differentiators
Persistent learning memory.
Adaptive Recall Engine inspired by the forgetting curve.
Learning Graph instead of flat progress tracking.
Explainable AI ("Why am I revising this?").
Gamified retention (reward remembering, not just studying).
Feature Prioritization
Must Have
Multi-agent architecture.
Persistent memory.
Adaptive Recall.
Dashboard.
Gamification.
Competitive Differentiator: Instead of presenting multiple unrelated AI features, EduPath demonstrates one cohesive AI system -The Cognitive Learning Engine (CLE).
Deployment.
Nice to Have
Voice input.
OCR notes.
Parent dashboard.
Out of Scope
Multiplayer.
Native mobile app.
Teacher portal.
Demo Philosophy

The demo should tell a story:

Student returns after several days.
EduPath remembers them.
EduPath predicts forgotten concepts.
Student recovers knowledge.
Dashboard updates in real time.
AI explains why it chose those questions.
